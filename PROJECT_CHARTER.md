# FoldingOS Project Charter

Version: 1.0

Status: Active

---

# Mission

FoldingOS is an open-source, purpose-built operating system designed
exclusively to maximize scientific contribution to Folding@home.

Its mission is to provide a secure, reliable, lightweight, and maintainable
platform that enables individuals and organizations to contribute computing
resources with minimal operational complexity.

FoldingOS is intentionally not a general-purpose Linux distribution.

---

# Vision

To become the simplest and most reliable operating system dedicated to
advancing Folding@home scientific research.

Users should be able to:

Download

↓

Flash

↓

Boot

↓

Configure

↓

Fold

without requiring extensive Linux knowledge or ongoing administration.

---

# Project Objectives

The primary objectives of FoldingOS are:

- maximize scientific contribution

- maximize operational reliability

- minimize administration

- minimize attack surface

- minimize unnecessary complexity

- provide reproducible builds

- support centralized fleet management

- remain understandable and maintainable

---

# Core Values

## Reliability

Reliable operation is more important than feature count.

---

## Simplicity

Simple solutions are preferred over clever solutions.

Complexity requires justification.

---

## Security

Security is a design requirement.

Secure defaults should always be preferred.

---

## Documentation

Documentation is part of the architecture.

Documentation precedes implementation.

---

## Reproducibility

Every release should be reproducible from source.

Engineering decisions should be transparent and documented.

---

## Maintainability

Future contributors should understand not only what exists,
but why it exists.

---

# Project Philosophy

FoldingOS should behave as an appliance.

Users should deploy it and allow it to perform its intended purpose with
minimal ongoing administration.

The operating system should remain focused on scientific contribution rather
than becoming a general-purpose computing platform.

---

# Non-Goals

FoldingOS is not intended to become:

- Desktop Linux

- Development workstation

- Media server

- NAS

- Gaming platform

- Browser platform

- Kubernetes platform

- Docker host

- General-purpose Linux distribution

Scope discipline is considered a project strength.

---

# Engineering Standards

Engineering decisions should prioritize:

1. Reliability

2. Security

3. Simplicity

4. Maintainability

5. Documentation

6. Reproducibility

7. Performance

8. Features

---

# Long-Term Vision

The long-term objective is to create an operating system that can reliably
operate unattended for extended periods while maximizing contribution to
Folding@home scientific research.

The project should remain:

- understandable

- transparent

- well documented

- professionally engineered

- welcoming to contributors

---

# Closing Statement

FoldingOS exists to demonstrate that focused engineering, thoughtful
architecture, and disciplined simplicity can produce software that is both
powerful and maintainable.

Every design decision should reinforce the project's mission of advancing
scientific research through reliable computation.
