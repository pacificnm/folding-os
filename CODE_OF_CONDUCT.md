# FoldingOS Code of Conduct

## Our Commitment

We are committed to fostering an open, respectful, and collaborative
community.

FoldingOS exists to support scientific research through thoughtful engineering
and open collaboration.

We welcome contributors of all backgrounds, experience levels, and technical
interests who share the project's goals.

---

# Our Standards

Examples of positive behavior include:

- treating others respectfully

- providing constructive feedback

- welcoming new contributors

- discussing technical disagreements professionally

- assuming good intentions

- focusing on evidence and engineering merit

- helping improve documentation

- helping improve code quality

---

# Unacceptable Behavior

Unacceptable behavior includes:

- harassment

- discrimination

- personal attacks

- intimidation

- threats

- abusive language

- deliberate misinformation

- trolling

- doxxing

- publishing private information without permission

Technical disagreement is acceptable.

Personal attacks are not.

---

# Engineering Culture

The FoldingOS project values:

- humility

- curiosity

- documentation

- transparency

- professionalism

- constructive criticism

We encourage thoughtful discussion and evidence-based decision making.

Being technically correct does not justify being disrespectful.

---

# Decision Making

Project decisions should be guided by:

- technical merit

- project philosophy

- maintainability

- reliability

- simplicity

rather than personal preference.

Healthy disagreement is an important part of engineering.

---

# Community Expectations

Contributors should:

- respect differing opinions

- remain open to feedback

- explain technical reasoning

- document important decisions

- prioritize project goals over personal preference

The objective is to build great software together.

---

# Enforcement

Project maintainers are responsible for maintaining a respectful and
productive environment.

Responses to unacceptable behavior may include:

- warnings

- removal of comments

- temporary restrictions

- permanent removal from project participation

depending on severity.

---

# Scope

This Code of Conduct applies to:

- project discussions

- GitHub issues

- pull requests

- documentation

- code reviews

- community communication channels

where project participants interact.

---

# Reporting

If you experience or witness behavior that violates this Code of Conduct,
please contact the project maintainers.

Reports should be handled respectfully and, where practical,
confidentially.

---

# Final Principle

Treat people with the same professionalism and respect that you expect from
others.

Good engineering is built not only on good code, but on good collaboration.
